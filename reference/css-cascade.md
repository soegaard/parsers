# CSS Cascade

Primary URL:

- <https://www.w3.org/TR/css-cascade-5/>

Why this document matters:

- It defines core concepts around declarations, importance, origins, and
  cascade-related syntax.
- It matters less for the first structural parser than CSS Syntax does, but it
  becomes relevant as soon as declarations are represented more precisely.

What to use it for in this repo:

- understanding declaration-level structure
- future handling of `!important`
- future support for cascade layers and related constructs

Implementation note:

- this is primarily a semantic and declaration-shape reference, not the main
  tokenization/parsing algorithm reference
